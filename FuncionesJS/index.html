Funciones en JS: El Misterio de los Argumentos

Recurso para la UTN: Paso de parámetros, gestión de memoria y tipos de objetos.

¿Valor o Referencia?

En JavaScript, no elegimos la sintaxis (como en C). El comportamiento depende del tipo de dato.

Paso por Valor (Primitivos)

Numbers, Strings, Booleans. Se crea una copia.

let n = 10;
function cambiar(val) {
    val = 20; // No afecta a 'n'
}
cambiar(n);
console.log(n); // Resultado: 10

Paso por Referencia (Objetos)

Objects, Arrays. Se pasa la dirección de memoria.

let obj = { x: 10 };
function cambiar(o) {
    o.x = 20; // ¡Afecta al original!
}
cambiar(obj);
console.log(obj.x); // Resultado: 20

¿Objeto Literal o Instancia de Clase?

Es común preguntarse de dónde "vienen" los objetos en JavaScript cuando no definimos una clase.

Objeto Literal

let auto = { marca: "Ford" };
  • ✅ ¿Es una instancia? Sí, de la clase global Object.
  • ✅ ¿Tiene clase? Su prototipo es Object.prototype.
  • 📌 Es ideal para agrupar datos de forma rápida (como un struct en C).

Instancia de Clase (ES6)

class Vehiculo { ... }
let auto = new Vehiculo();
  • ✅ ¿Es una instancia? Sí, de la clase Vehiculo.
  • ✅ Herencia: También es instancia de Object por herencia.
  • 📌 Se usa cuando necesitamos comportamiento (métodos) y una estructura definida.
Nota para la clase: Independientemente de cómo se cree (literal o con new), en JavaScript todos los objetos se guardan en el Heap y se acceden mediante una referencia en el Stack.

Modelo de Memoria en JS

Para entender por qué los objetos se "pasan por referencia", debemos mirar dónde guarda JS cada cosa.

STACK Memoria Estática (Pila)

Almacena datos de tamaño fijo y rápido acceso.

  • Variables primitivas (Number, Boolean).
  • Referencias (punteros a objetos en el Heap).
  • Stack Frames de funciones (como en C).

HEAP Memoria Dinámica (Montón)

Almacena datos de tamaño variable y complejo.

  • Objetos ({...}).
  • Arrays ([...]).
  • Funciones.
  • Gestionado por el Garbage Collector (sin free() manual).

Visualización Mental

n: 10
obj: #001
➜
HEAP

{ marca: "Ford", ... }

Cuando pasamos 'obj' a una función, copiamos el #001 (la dirección), no el contenido del Heap.

La Trampa de la Reasignación

"Si reasignas el objeto completo dentro de la función, rompes el vínculo con la dirección de memoria original."

CASO A: Mutar Propiedad

let auto = { marca: "Ford" };

function mutar(obj) {
    obj.marca = "Fiat"; 
}

mutar(auto);
console.log(auto.marca); // "Fiat"

Afecta al original (mismo #ID)

CASO B: Reasignar Objeto

let auto = { marca: "Ford" };

function reasignar(obj) {
    obj = { marca: "Fiat" }; 
}

reasignar(auto);
console.log(auto.marca); // "Ford"

NO afecta (nueva dirección en Stack)
